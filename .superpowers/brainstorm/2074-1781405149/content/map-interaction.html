地图交互 — 点击位置后的信息面板

核心问题：地图上的交互点需要承载哪些信息？以下是三种不同信息深度的设计方案

A

轻量卡片 — 概要信息 + 跳转

点击位置 → 弹出小型浮动卡片，显示：名称、类型、简短描述、关联人物数、关联事件数。点击展开 → 跳转至人物/时间线页面查看详情。类似地图 App 点击 POI 的效果。

B

侧边面板 — 同一位置的全景信息

点击位置 → 侧边栏滑出，包含：详细描述、当前在此的人物列表（可筛选时间）、发生于此的事件时间线、所属势力/体系标签。所有数据在一个面板内，无需跳转。

C

深度节点 — 多系统关系网

点击位置 → 展开以该位置为中心的关系图谱：展示「谁来过这里 → 发生了什么事 → 涉及什么体系/元素 → 时间线上排在什么位置」。直接在地图上呈现节点关联线。
