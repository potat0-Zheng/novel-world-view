## 二、核心设计决策

### 2.1 渲染方案

| 决策 | 选择 | 理由 |
|------|------|------|
| 主视角 | 2.5D 等距（摄像机倾角 ~60°） | 信息密度高，开发成本适中 |
| 备选视角 | 2D 俯视 | 一键切换，快速全局浏览 |
| 视角切换 | 摄像机平滑动画（0.8s） | 参考地图 App 体验 |
| 交叉过渡 | 倾角降至 15°-20° 时开始淡入图标/淡出模型 | 避免 0° 突变 |
| 3D 模型 | Low-Poly 风格，GLTF/GLB 格式 | 个人项目可维护，AI 生成友好 |

### 2.2 地图编辑

| 决策 | 选择 |
|------|------|
| 数据结构 | 基于单元格（cell-based），N×N 网格 |
| 创建流程 | **先 2D 网格设定（GridSetup）→ 确认后进入 2.5D 编辑器** |
| 视觉渲染 | 数据是格，视觉是有机的——相邻同类格自动融合 |
| 陆地呈现 | **3D 长方体块**（灰白色 `#c8c0b0`，高 0.3 单位），非平面顶点着色 |

### 2.3 三层图层架构

```
L1 🟩 基础地形 — 海洋/大陆，全覆盖互斥，GridSetup 阶段设定
L2 🌿 自然地形 — 山脉/森林/平原，仅大陆格上，互斥（一格一种）
L3 🏗 建筑模型 — 6 种 demo 模型，仅大陆格上，可叠加
```

## 三、技术方案

### 3.1 最终选择：React + React Three Fiber (R3F)

| 层面 | 技术 |
|------|------|
| 框架 | React 18 + TypeScript |
| 3D 渲染 | Three.js + @react-three/fiber + @react-three/drei |
| 状态管理 | Zustand（单一数据源 + 空间/时间索引模式） |
| 路由 | React Router |
| 构建 | Vite |
| 存储 | localStorage（demo）；IndexedDB（后续） |

### 3.2 数据引擎设计

采用「单一数据源 + 索引」模式：
```typescript
interface WorldData {
  config: WorldConfig;
  cells: Record<CoordKey, CellData>;       // 网格单元
  entities: Record<string, Entity>;         // 人物/事件/势力
  locationIndex: Record<CoordKey, string[]>; // 空间索引
  timeIndex: Record<string, string[]>;       // 时间索引
}
```

### 3.3 远期备选

Svelte + Threlte 作为体积优化备选（目前社区太小不推荐）。

## 五、Bug 修复记录

### 5.1 模型不显示（首次）

**症状：** GridSetup 确认后进入 2.5D 页面，看不到任何地形。

**根因：**
1. CameraController 渲染了独立的 `<perspectiveCamera>`，初始位置 (0,0,0)，首帧无内容
2. GridSetup 用 `initWorld` + 100 次 `setCellL1` 逐格写入，状态更新过于频繁

**修复：**
1. CameraController 改用 `useThree()` 获取 Canvas 默认摄像机，渲染前直接设好 2.5D 位置
2. GridSetup 改用 `loadWorld` 一次性构建完整 WorldData 写入

**Commit:** `0ecbdf0`

### 5.2 2D/2.5D 标签反向 + 无法绘制 L2/L3

**症状：**
1. ViewToggle 按钮标签显示目标模式而非当前模式（在 2.5D 时显示"2D"）
2. ClickHandler 只做选中，不根据编辑器模式执行绘制

**修复：**
1. ViewToggle 改为显示当前模式
2. ClickHandler 根据 `editorMode` 分派：
   - `l1_paint` → `setCellL1`
   - `l2_paint` → `setCellL2`（仅大陆格）
   - `l3_place` → `setCellModel`（仅大陆格+已选模型）
   - `view` → `setSelectedCellKey`（仅选中）

**Commit:** `45a4f88`
